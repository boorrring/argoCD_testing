kubectl get namespaces
kubectl create namespace <namespace-name>
kubectl delete namespace <namespace-name>
kubectl describe namespace <namespace-name>


kubectl apply -f <filename.yaml>: Creates or updates resources defined in a YAML or JSON file.
kubectl create -f <filename.yaml>: Creates resources from a file.    (kubectl create -f folder_name/)
kubectl delete -f <filename.yaml>: Deletes resources defined in a file.


kubectl get <resource_type>:
    Lists resources of a specific type (e.g., pods, deployments, services, nodes).
kubectl get all: Lists all common resources in the current namespace. 

kubectl describe <resource_type> <resource_name>:
    Displays detailed information about a specific resource, useful for troubleshooting.
kubectl logs <pod_name>:
    Displays the logs of a running pod.

kubectl top pod/node: 
    Shows resource usage (CPU, memory) for pods or nodes (requires the Metrics Server to be installed).




kubectl port-forward service/my-service 5000   // forwarding ports inside the kubernetes to
            the out machine port. Here the port on which app is running inside the kube will be
            forwarded to the port "5000" of the outer machine.
